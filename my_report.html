1. 目的 (Introduction)

本研究では、ワーキングメモリ負荷が情報の保持および更新プロセスに及ぼす影響について検討する。ワーキングメモリ（作業記憶）とは、情報の保持と処理を同時に行う一時的な認知システムである（Baddeley, 1992,(Baddeley 1992)）。 このシステムの容量には限界があり、処理すべき情報量や複雑さ（認知負荷）が増大すると、パフォーマンスが低下することが知られている。ワーキングメモリの評価には、提示される刺激がn個前のものと一致しているかを判断する「n-Back課題」が広く用いられてきた（Kirchner, 1958,(Kirchner 1958)）。nの数（負荷レベル）が増えるほど、情報の維持だけでなく、古い情報の消去と新しい情報の格納という「アップデート（更新）」の負荷が急速に高まる（Jonides et al., 1997,(Jonides et al. 1997)）。本実験の仮説は以下の2点とした。 仮説1（正答率）：1-Back条件に比べ、2-Back条件の方が課題の正答率（ヒット率と正当棄却率の総和）が有意に低下する。 仮説2（反応時間）：1-Back条件に比べ、2-Back条件の方が、正当試行（ヒットおよび正当棄却）における平均反応時間が有意に遅く（長く）なる。

2. 方法 (Method)

参加者

２０名のデータを収集した。

装置と材料

オンライン実験プラットフォーム（jsPsych）を使用し、参加者個人のPC、タブレット、スマホのWebブラウザ上で実施された。

手続き

実験で参加者は1-Back条件および2-Back条件の2セッションを行った。 1-Back条件：現在呈示されている文字が、「1つ前」に呈示された文字と同一であるか否かを判断した。 2-Back条件：現在呈示されている文字が、「2つ前」に呈示された文字と同一であるか否かを判断した。

3. 結果 (Results)

以下の通り、対応のあるt検定を行った。 1-Back条件：現在呈示されている文字が、「1つ前」に呈示された文字と同一であるか否かを判断した。 2-Back条件：現在呈示されている文字が、「2つ前」に呈示された文字と同一であるか否かを判断した。 記述統計を算出した結果、正答率の平均値および標準偏差は、1-Back条件において \(M = 83.33\%\)（\(SD = 28.87\%\)）、2-Back条件において \(M = 66.67\%\)（\(SD = 57.74\%\)）であった。

対応のある \(t\) 検定の結果、条件間における正答率の有意な差は認められなかった（\(t(2) = 0.38\), \(p = .742\), \(95\%\) 信頼区間 \([-1.73, 2.06]\)）。

# 2. データの読み込み ----------------------------------------------------------
data_file <- "clean_data.csv"

# ファイルが存在しない場合の警告処理と自動生成（デモ用）
if (!file.exists(data_file)) {
  message("警告: 'clean_data.csv' が見つかりません。")
  message("コードの動作確認用に、一時的なN-Back課題のダミーデータを自動生成して処理を続行します。")
  
  # 実データ（target / non-target）の仕様に合わせてダミーを生成
  set.seed(42)
  dummy_raw <- expand.grid(
    subject_id = paste0("Sub_", sprintf("%02d", 1:30)),
    condition = c("non-target", "target"),
    trial = 1:20
  ) %>%
    mutate(
      rt = rnorm(n(), mean = 500, sd = 100),
      # target条件は少し難しく正答率がばらつく（平均80%）、non-targetは簡単（平均95%）
      correct = if_else(condition == "non-target",
                        rbinom(n(), 1, 0.95) == 1,
                        rbinom(n(), 1, 0.80) == 1)
    ) %>%
    select(subject_id, condition, rt, correct)
  
  write_csv(dummy_raw, data_file)
  message("-> 一時的な 'clean_data.csv' を作成しました。\n")
}

raw_data <- read_csv(data_file, show_col_types = FALSE)

# 3. データ集計（正答率の2段階算出と標準誤差の算出） ---------------------------

# 段階1: 参加者 (subject_id) ごと、各条件 (condition) ごとの平均正答率を算出
sub_summary <- raw_data %>%
  # correct列を論理値(TRUE/FALSE)に強制変換
  mutate(correct_logical = as.logical(correct)) %>%
  group_by(subject_id, condition) %>%
  summarize(
    sub_accuracy = mean(correct_logical, na.rm = TRUE),
    .groups = "drop"
  )

# 段階2: 全参加者における条件ごとの「全体の平均正答率」と「標準誤差（SE）」を算出
final_summary <- sub_summary %>%
  group_by(condition) %>%
  summarize(
    mean_accuracy = mean(sub_accuracy, na.rm = TRUE),
    n = n(),
    sd_accuracy = sd(sub_accuracy, na.rm = TRUE),
    se_accuracy = sd_accuracy / sqrt(n),
    .groups = "drop"
  )

# 集計結果をコンソールに表示
cat("--- 条件ごとの集計結果 (参加者内平均正答率に基づく) ---\n")
--- 条件ごとの集計結果 (参加者内平均正答率に基づく) ---
print(final_summary)
